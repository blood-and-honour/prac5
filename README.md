# Практическая работа 5 – Анимации с GSAP и Framer Motion

## 🚀 Запуск проекта

```bash
npm install
npm run dev
```

## 🎯 Что сделано

В проект добавлены **интерактивные анимации** с использованием двух библиотек: **GSAP** и **Framer Motion**. Анимации присутствуют на всех страницах и делают интерфейс живым, отзывчивым и понятным для пользователя.

---

## 🧩 Анимации GSAP

**GSAP** использован для сложных, последовательных анимаций, где важен точный контроль времени и координат.

### Главная страница (`Home.tsx`)
- **Появление заголовков и карточек** – плавное появление с задержками (stagger), выезд снизу, масштабирование.
- **Hover на карточках** – карточка подлетает вверх (+8px) и увеличивается (scale 1.02).
- **Кнопка «Подписаться»** – при наведении увеличивается, при клике – пульсация (увеличение/уменьшение) и мигание жёлтым цветом (обратная связь об успешной подписке).

### Страница услуг (`Products.tsx`)
- **Появление карточек** – выезд слева с задержкой по очереди (stagger).
- **Hover на карточках** – аналогично главной: подлёт и увеличение.

### Страница «О компании» (`About.tsx`)
- **Stagger-анимация текста** – каждый абзац и пункт списка появляется последовательно с задержкой 0.1 сек.

### Страница контактов (`Contacts.tsx`) и профиль (`Profile.tsx`)
- **Появление блока формы** – плавное появление с масштабированием (scale от 0.95 до 1).
- **Кнопки «Отправить» / «Сохранить»** – при наведении увеличение, при клике пульсация + жёлтая вспышка + всплывающее уведомление.

---

## ✨ Анимации Framer Motion

**Framer Motion** использован для декларативных анимаций, связанных с состоянием (появление/исчезновение), а также для простых интерактивных эффектов.

### Главная страница (`Home.tsx`)
- **Уведомление «Вы успешно подписались»** – появляется плавно (затухание + выезд) под кнопкой, через 2 секунды исчезает.
- **Кнопка «Узнать историю компании»** – появляется с задержкой 0.5 сек (анимация масштаба и прозрачности), при наведении затемняется и увеличивается.
- **Блок истории компании** – при клике на кнопку плавно появляется (выезд сверху + затухание), при повторном клике – исчезает. Использован `AnimatePresence` для анимации удаления из DOM.

### Страницы контактов и профиля
- **Всплывающие уведомления** («Сообщение отправлено!», «Данные сохранены!») – аналогично главной: плавное появление/исчезновение, не ломает вёрстку (фиксированная высота контейнера).

## 🔍 Сравнение GSAP и Framer Motion

| Критерий | GSAP | Framer Motion |
|----------|------|----------------|
| **Подход** | Императивный (через `useEffect` + `ref`) | Декларативный (прямо в JSX) |
| **Сложные анимации** | Отлично (таймлайны, stagger, yoyo) | Средне (лучше для простых) |
| **Анимация появления/исчезновения** | Требует ручного управления | Идеально (`AnimatePresence`) |
| **Анимация при клике/hover** | Хорошо, но нужно сбрасывать | Просто (`whileHover`, `whileTap`) |
| **Интеграция с React** | Нужны `ref` и очистка | Из коробки |
| **Производительность** | Высокая | Высокая |

### Что выбрали и почему

- **GSAP** – для анимаций, где нужен чёткий контроль времени, последовательность (stagger), сложные эффекты (пульсация с yoyo). Он даёт больше возможностей для «тяжёлых» анимаций.
- **Framer Motion** – для анимаций, зависящих от состояния (открыть/закрыть), для всплывающих уведомлений, для простых hover/click-эффектов. Код становится короче и понятнее.

**Итог:** оба инструмента отлично дополняют друг друга. В проекте они используются там, где каждый силён. Это позволяет сделать интерфейс современным, плавным и понятным пользователю.
